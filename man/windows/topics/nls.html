nls – Nicels Command-Line Utility

nls is a modern, cross-platform directory listing tool that serves as a drop-in replacement for ls. It provides richly formatted output with icons, color themes, Git status integration, and configurable layouts. Use nls to list files and directories with more information and better visuals than traditional ls.

Usage

nls [OPTIONS] [PATH...]

If no PATH is given, nls lists the current directory.

Command-Line Options

Layout

  • -l, --long – long listing format.
  • -1, --one-per-line – one entry per line.
  • -x – list across, by rows.
  • -C – list by columns (default).
  • --format=<WORD> – vertical|horizontal|long|single-column|comma.
  • --header – force header (dir name/column labels in long format).
  • -m – comma-separated stream output.
  • -T, --tabsize COLS – tab stops every COLS (default 8).
  • -w, --width COLS – wrap to width (0 = no limit).
  • --tree[=DEPTH] – recursive tree view; optional depth (0 = unlimited).
  • --report[=short|long] – summary report (long if omitted).
  • --zero – NUL-terminate entries.

Filtering

  • -a, --all – include dotfiles.
  • -A, --almost-all – include dotfiles except . and ...
  • -d, --dirs – only directories.
  • -f, --files – only files.
  • -B, --ignore-backups – ignore *~.
  • --hide PATTERN – hide names matching PATTERN.
  • -I, --ignore PATTERN – ignore names matching PATTERN.

Sorting

  • -t – sort by mtime (newest first).
  • -S – sort by size (largest first).
  • -X – sort by extension.
  • -U – unsorted (directory order).
  • -r, --reverse – reverse sort order.
  • --sort=<name|time|size|extension|none>
  • --sd, --sort-dirs, --group-directories-first
  • --sf, --sort-files
  • --df, --dots-first

Appearance

  • -b, --escape – C-style escapes for nongraphic chars.
  • -N, --literal – literal names (no quoting).
  • -Q, --quote-name – quote names with double quotes.
  • --quoting-style=<WORD> – literal|locale|shell|…|c|escape.
  • -p – append / to directories.
  • --indicator-style=<slash|none>
  • --no-icons, --without-icons
  • --no-color / --color=auto|always|never
  • --theme NAME, --light, --dark
  • -q, --hide-control-chars / --show-control-chars
  • --time-style=FORMAT, --full-time
  • --hyperlink

Information

  • -i, --inode
  • -o (long without group), -g (long without owner), -G, --no-group
  • -n, --numeric-uid-gid
  • --bytes, --non-human-readable, -s, --size, --block-size=SIZE
  • -L, --dereference
  • --gs, --git-status
  • --perf-debug
  • --copy-config

Environment

  • NLS_DATA_DIR – path to themes/icons config.
  • LS_COLORS – color rules (GNU dircolors style).
  • NO_COLOR – disable color.
  • TIME_STYLE – default time style.
  • QUOTING_STYLE – default quoting style.
  • POSIXLY_CORRECT – stricter POSIX behavior in some cases.

Examples

# Long + all + git + header + report
nls -la --gs --header --report=long

# Tree up to depth 2
nls --tree=2 ~/projects

# Only directories (long)
nls -dl

# Copy default config
nls --copy-config
